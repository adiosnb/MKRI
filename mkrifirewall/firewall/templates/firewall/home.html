nft based firewall for MKRI

Firewall application is implemented as a web based interface to nftables framework, which is a part of Netfilter subsystem.

Primary programming language used in backend is Python and Shell, where frontend is implemented with Django, HighCharts, Bootstrap, Ajax and similar frameworks.

Django is used to display queries from the nft userspace utility, that controls Netfilter's packet filtering based on newer implementation of firewall in form of nftables.

Due to keeping the test environment simple enough, firewall is implemented in netfilter's input and output hooks, instead of forward one. This alteration allows user to observe incoming and outgoing traffic from network endpoint, instead of a need to create real network with typically transparent forwarding firewall.

Firewall

User has full control over the nft configuration, ranging from tables to specific rules. Application offers some preconfigured firewall rules to include statistics for well-known encrypted traffic.

Test environment

Vagrant is the chosen tool to create replicable virtualized environment for the ease of testing and presenting the application. It provisions libvirt based virtual machine with specified usecase, either generator or firewall itself.

Generator VM is used to generate traffic with iperf3 framework, to verify functionality of application. It is connected directly to the firewall VM, where one endpoint of IPsec tunnel is present as well.

Firewall VM instance is running the nft ruleset itself, which is doing the actual traffic filtering. Other endpoint of IPsec tunnel is located here, along with the webserver instance generated from Django.
